# 阶段 4：质量保障、导出回收站与运维补齐

## 1. 阶段目标

本阶段用于补齐正式项目应有的外围能力，使系统更适合长期使用和维护。

本阶段完成后，应达到以下结果：

- 题库支持导出
- 删除操作更安全
- 项目具备基础测试能力
- 配置、安全、日志和备份更加规范

## 2. 本阶段边界

本阶段要做：

- 导出能力
- 软删除与回收站
- 测试补齐
- 环境变量配置
- 日志和备份

## 3. 导出能力

### 3.1 导出格式

建议优先支持：

- JSON
- Markdown
- CSV

### 3.2 导出接口

建议新增：

- `GET /api/banks/:id/export?format=json`
- `GET /api/banks/:id/export?format=markdown`
- `GET /api/banks/:id/export?format=csv`

### 3.3 导出内容结构

JSON 建议保留完整结构：

- 题库元信息
- 题目
- 选项
- 答案
- 解析

Markdown 建议适合阅读和分享。

CSV 建议适合批量编辑和二次导入。

## 4. 回收站与删除策略

### 4.1 为什么需要软删除

当前直接删除风险较高，尤其在导入错误、误删题库、误删目录时不可恢复。

### 4.2 实现方案

为以下表增加：

- `deleted_at`

适用对象：

- `folders`
- `banks`
- `questions`

### 4.3 删除流程

建议逻辑：

1. 默认软删除
2. 回收站中可恢复
3. 回收站中可永久删除

### 4.4 恢复逻辑

恢复时要校验：

- 原父目录是否仍存在
- 原题库是否仍存在
- 名称是否冲突

## 5. 测试体系

### 5.1 测试优先级

必须优先覆盖：

- 导入解析器
- 去重逻辑
- 删除与恢复逻辑
- 导出逻辑

### 5.2 推荐目录

```text
tests/
  unit/
  integration/
  fixtures/
```

### 5.3 单元测试建议

覆盖：

- 文本清洗
- 题型识别
- 答案提取
- 哈希生成

### 5.4 集成测试建议

覆盖：

- 导入到入库完整链路
- 删除题库后的结果
- 恢复逻辑
- 导出结果结构

## 6. 配置与安全

### 6.1 环境变量

建议新增：

- `.env.example`

内容建议：

```env
PORT=3007
JWT_SECRET=replace_this
DB_PATH=./db/database.sqlite
AI_TIMEOUT_MS=30000
```

### 6.2 安全改造

要完成：

- 去除硬编码密钥
- 增加强密码和输入校验
- 统一错误返回
- 强化资源归属检查

## 7. 日志与备份

### 7.1 日志建议

至少记录：

- 登录失败
- 导入任务
- AI 调用失败
- 删除和恢复操作

### 7.2 备份建议

SQLite 可先采用轻量策略：

- 每次迁移前备份
- 每次大批量导入前备份
- 保留最近 N 份副本

## 8. 文档与交接

本阶段还应补：

- `README.md`
- 本地启动说明
- 数据迁移说明
- 导入格式说明

## 9. 验收标准

本阶段完成后，应满足：

- 题库可导出
- 删除可恢复
- 关键逻辑有基础测试
- 配置和密钥不再硬编码
- 有最基本的日志与备份策略

## 10. 建议拆分任务

1. 增加导出接口
2. 增加软删除字段和回收站接口
3. 增加恢复和永久删除逻辑
4. 建立测试目录和测试样例
5. 增加 `.env.example`
6. 增加日志和备份脚本

## 11. 阶段完成标志

当下面条件满足时，可认为阶段 4 完成：

- 项目已经不仅是“能开发”，而是具备基本的长期维护能力
