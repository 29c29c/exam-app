# 阶段 2：前端模块化与资源管理增强

## 1. 阶段目标

本阶段目标是把当前单文件前端和简化资源管理能力，升级为更易维护、可扩展的正式结构。

本阶段完成后，应达到以下结果：

- 前端从单个 `index.html` 中拆分出来
- 仪表盘支持更完整的文件夹和题库管理
- 目录导航更加清晰
- 题库管理不再只靠“进入/删除”两个动作

## 2. 本阶段边界

本阶段要做：

- React 组件拆分
- API 调用层拆分
- 仪表盘结构重构
- 文件夹/题库重命名、移动
- 面包屑与目录树
- 题库统计信息展示

本阶段不做：

- AI 任务系统
- 回收站
- 导出
- 完整测试体系

## 3. 前端结构重构

### 3.1 推荐目录

```text
src/
  app/
    App.jsx
  pages/
    DashboardPage.jsx
    QuizPage.jsx
  components/
    Auth/
    Dashboard/
    Import/
    Quiz/
    Common/
  api/
    client.js
    folders.js
    banks.js
    questions.js
  styles/
```

### 3.2 拆分顺序

建议按以下顺序拆：

1. 请求封装
2. 布局壳组件
3. 仪表盘组件
4. 做题组件
5. 弹窗组件

### 3.3 请求层实现

建议把当前零散 `request()` 调用抽为：

- `api/client.js`
- `api/auth.js`
- `api/folders.js`
- `api/banks.js`
- `api/questions.js`
- `api/import.js`

目标：

- 避免每个组件都自己拼接口
- 统一处理 token 和错误

## 4. 仪表盘重构

### 4.1 面包屑导航

当前仅靠 `currentFolder` 状态回退，不适合多层目录。

建议实现：

- 后端提供当前节点的祖先路径
- 前端维护 `folderPath`
- 点击任意层级可跳转

### 4.2 文件夹树视图

建议支持两种视图：

- 当前层列表视图
- 左侧或弹出式目录树视图

第一阶段实现可简化为：

- 获取平铺目录
- 前端转树

### 4.3 题库卡片增强

每个题库卡片建议展示：

- 名称
- 题目数
- 收藏数
- 最近更新时间
- 未解析数量

建议新增操作菜单：

- 做题
- 收藏夹
- 重命名
- 移动
- 批量 AI 解析
- 导出
- 删除

## 5. 文件夹与题库管理能力

### 5.1 文件夹能力

新增接口建议：

- `PATCH /api/folders/:id`
- `PATCH /api/folders/:id/move`

要支持：

- 修改名称
- 调整父目录
- 阻止移动到自身子树

### 5.2 题库能力

新增接口建议：

- `PATCH /api/banks/:id`
- `PATCH /api/banks/:id/move`
- `GET /api/banks/:id/stats`

要支持：

- 重命名
- 移动到其他目录
- 获取统计信息

### 5.3 批量操作

建议增加选择模式：

- 批量删除
- 批量移动

实现方式：

- 列表进入多选模式
- 勾选多个题库
- 执行同一动作

## 6. 前端交互升级

### 6.1 删除交互

建议改为：

- 删除前显示影响范围
- 删除后给出明确反馈

例如删除文件夹时提示：

- 子文件夹数量
- 题库数量
- 题目数量

### 6.2 空状态与提示

当前空状态只有“暂无内容”。

建议补：

- 没有文件夹时的引导
- 没有题库时的导入入口
- 搜索为空时的提示

### 6.3 表单交互

建议统一处理：

- 名称重复校验
- 非法字符提示
- 提交中状态
- 成功反馈

## 7. 后端支撑改造

本阶段虽然偏前端，但后端需要补足资源管理接口。

建议新增服务：

- `folder-service.js`
- `bank-service.js`
- `stats-service.js`

### 7.1 统计信息来源

题库统计建议至少包含：

- `question_count`
- `bookmark_count`
- `analysis_count`
- `updated_at`

实现方法有两种：

- 实时查询
- 冗余统计字段 + 写入时更新

建议先用实时查询，结构稳定后再考虑冗余优化。

## 8. 验收标准

本阶段完成后，应满足：

- 前端主逻辑不再全部堆在一个 HTML 文件里
- 用户可重命名和移动文件夹/题库
- 多层目录导航清晰可用
- 题库卡片可以显示基础统计
- 仪表盘可作为稳定管理页使用

## 9. 建议拆分任务

1. 建立 `src/` 目录结构
2. 拆分 API 请求层
3. 拆分 Dashboard 与 Quiz 组件
4. 增加面包屑导航
5. 增加题库重命名和移动
6. 增加文件夹重命名和移动
7. 增加题库统计展示

## 10. 阶段完成标志

当下面条件满足时，可认为阶段 2 完成：

- 前端已从单 HTML 结构中解耦
- 资源管理具备“增删改移”完整能力
- 目录导航和题库信息足够支撑日常使用
