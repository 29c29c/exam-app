# AI 智能题库

> [!WARNING]
> 这是一个 `100% vibecoding` 项目。
> 其中大约 `90%` 由 `Gemini` 生成，`10%` 由 `Codex` 补充和整理。
> 这个项目默认按“自用工具”定位开发，`没有可靠的长期维护能力`，也`不保证架构稳定性、代码一致性、后续兼容性或生产可维护性`。
> 如果你准备把它用于正式环境、多人协作、商业用途或长期运营，请先自行评估风险，最好把它当作原型或私人工具，而不是可承诺维护的产品。

一个基于 `Node.js + Express + SQLite + React(浏览器直跑)` 的自定义题库管理与学习工具。

它主要解决的是这几类很现实的痛点：

- 题库散落在微信、文档、聊天记录、PDF 摘抄里，难以统一管理
- 想自己维护题库，但很多刷题软件不支持真正自定义、多端同步和长期积累
- 做题时希望有 AI 帮忙解释题目，而不是只看一个冷冰冰的标准答案
- 想把错题、收藏题、学习进度沉淀下来，而不是每次重新整理

这个项目目前更适合：

- 自己维护考研 / 考公 / 法考 / 教资 / 课程复习题库
- 从文档或 AI 输出内容中整理题目并导入
- 在电脑和手机浏览器里使用自己的题库
- 用 AI 辅助理解题目、整理解析、沉淀错题和收藏

## 项目现状

- 当前状态：`可用的自用项目`

- 题库导入预解析与确认导入
- 常见 AI Markdown / 纯文本题库导入

## 适合解决的场景

### 1. 自定义多端题库

你可以自己建文件夹、题库，把零散题目导入到一个统一系统中。只要服务部署在你自己的机器或服务器上，电脑和手机浏览器都能访问。

### 2. AI 解析辅助学习

不会做的题、看不懂的题、答案太简陋的题，可以直接调用 AI 生成解释，降低“只知道对错、不知道为什么”的痛点。

### 3. 错题与收藏沉淀

做题过程中可以收藏、标记学习状态，后续按“已掌握 / 待复习 / 学习中”继续刷题，不用每次重新整理。

- `Node.js 20.19.6`
- `npm 10+`
- `sqlite3 5.1.6`

本地如果你使用 `nvm`，推荐：

```bash
nvm install 20.19.6
nvm use 20.19.6
```

## 环境变量

参考文件见 [`.env.example`](./.env.example)。
常用配置如下：

| 变量名 | 默认值 | 说明 |
| --- | --- | --- |
| `PORT` | `8888` | 服务监听端口 |
| `JWT_SECRET` | 无 | 登录签名密钥，生产环境必须修改 |
| `DB_PATH` | `./db/database.sqlite` | SQLite 数据库文件路径 |
| `AI_TIMEOUT_MS` | `30000` | AI 请求超时时间 |
| `AI_CONFIG_SECRET` | 无 | AI 配置加密 / 签名相关密钥 |
| `CORS_ORIGIN` | 空 | 允许跨域的域名，多个用逗号分隔 |
| `COOKIE_SECURE` | `false` | 本地 http 调试可设 `false`，生产 https 建议 `true` |

推荐先复制：

```bash
cp .env.example .env
```

一个适合本地开发的示例：

```env
PORT=8888
JWT_SECRET=change_me_for_local_dev
DB_PATH=./db/database.sqlite
AI_TIMEOUT_MS=30000
AI_CONFIG_SECRET=change_me_too
CORS_ORIGIN=
COOKIE_SECURE=false
```

## 本地启动教程

### 1. 克隆并进入项目

```bash
git clone <your-repo-url>
cd exam-app
```

### 2. 切换到指定 Node 版本

```bash
nvm use 20.19.6
node -v
```

### 3. 安装依赖

```bash
npm install
```

### 4. 配置环境变量

```bash
cp .env.example .env
```

### 5. 如果本机 `sqlite3` 编译有问题，重建原生模块

```bash
export npm_config_cache="$PWD/.npm-cache"
export npm_config_devdir="$PWD/.node-gyp"
npm rebuild sqlite3
```

如果仍失败，再试：

```bash
npm rebuild sqlite3 --build-from-source
```

### 6. 启动服务

```bash
npm start
```

默认访问地址：

- [http://localhost:8888](http://localhost:8888)

## 服务器部署教程

下面给的是一个“个人服务器 / 自用 VPS”风格的最小部署流程。

### 1. 安装 Node 20.19.6

如果服务器用 `nvm`：

```bash
nvm install 20.19.6
nvm use 20.19.6
nvm alias default 20.19.6
```

### 2. 拉取项目并安装依赖

```bash
git clone <your-repo-url>
cd exam-app
npm install
```

### 3. 配置 `.env`

```bash
cp .env.example .env
```

生产环境至少建议修改：

- `JWT_SECRET`
- `AI_CONFIG_SECRET`
- `COOKIE_SECURE=true`
- `CORS_ORIGIN=https://your-domain.com`
- `DB_PATH` 指向稳定磁盘路径

### 4. 启动服务

先直接验证能否启动：

```bash
npm start
```

如果你想长期挂着跑，建议自行配 `pm2` 或 `systemd`。这个仓库目前没有内置部署脚本，所以部署层面还是“自己管”。

### 5. 反向代理

推荐自行使用 `Nginx` 或其他反向代理，把外网域名转发到：

- `http://127.0.0.1:8888`

同时建议你自己补：

- HTTPS
- 访问日志
- 错误日志
- 定期数据库备份
- 基本防火墙规则

## 软件使用教程

### 1. 注册 / 登录

- 首次使用需要注册账号
- 系统初始化时会自动生成一个默认邀请码
- README 中当前示例邀请码写为 `VIP888`

### 2. 建文件夹和题库

- 先建文件夹或直接建题库
- 支持多级目录
- 支持重命名、移动、删除和回收站恢复

### 3. 导入题库

点击题库导入后，把题目文本粘进去，系统会先做预解析，再让你确认导入。

目前支持：

- 纯文本题库
- 常见 AI 生成的 Markdown / 文本题库
- 单选题
- 多选题
- 基础判断题识别

当前对 `简答题` 的支持还不完整，后续会继续补。

### 4. 做题与管理学习状态

- 进入题库后开始刷题
- 可查看答案和解析
- 可收藏题目
- 可标记 `学习中 / 已掌握 / 待复习`
- 可按关键词、题型、学习状态等筛选

### 5. 使用 AI 解析

- 在页面中为当前账号配置你自己的 AI Key
- 支持单题 AI 解析
- 支持批量 AI 解析

说明：

- AI Key 会按当前账号加密保存在服务器
- 前端不会长期保存明文 Key

### 6. 导出与回收站

- 题库可导出为 `JSON / Markdown / CSV`
- 删除的文件夹、题库、题目会进入回收站
- 可以恢复，也可以永久删除

## 当前题库导入能力

支持粘贴纯文本题库，也支持粘贴常见 AI 生成的 Markdown 题库内容。

推荐纯文本格式：

```text
1. 马克思主义认为？
A. 选项A
B. 选项B
答案：B

2. 关于反映、信息和选择的关系，下列论断正确的是？
A. 论断A
B. 论断B
C. 论断C
答案：A,C
```

常见可兼容的 AI Markdown 风格包括：

- `## 第1题`
- `### 1.`
- `题目 1`
- `A.`、`A)`、`A：`
- `答案 / 参考答案 / 正确答案 / Answer`
- `解析 / 答案解析 / Explanation / Analysis`
- 标题、分隔线、列表、引用、代码块包裹的题目文本


## 暂时的自用 AI 处理提示词参考

> [!NOTE]
> 下面这份提示词是我当前“临时自用”的整理模板，`还没有按新版 AI 识别题库能力更新`。
> 它更适合把原始材料先整理成“规则导入更容易识别”的格式，再粘贴进系统。

```text
你是题库整理助手。请把我提供的原始资料整理成适合导入题库系统的纯文本格式，严格遵守以下要求：

1. 只输出整理后的题目内容，不要输出任何解释、前言、总结、提示语。
2. 每道题使用如下结构：
   1. 题干
   A. 选项A
   B. 选项B
   C. 选项C
   D. 选项D
   答案：A
   解析：xxx
3. 如果是多选题，答案统一写成：答案：A,C
4. 如果是判断题，答案统一写成：答案：对 或 答案：错
5. 不要使用 Markdown 表格。
6. 不要使用代码块包裹。
7. 不要省略题号。
8. 不确定答案时，答案留空，但保留“答案：”这一行。
9. 如果原文里没有解析，可以省略“解析：”这一行。
10. 不要改写题目本意，不要补充你自己的知识。

请开始整理以下内容：
```

## 已知限制

- 这是自用项目，不是规范化产品
- 前端仍然是偏轻量的浏览器直跑方案
- AI 调用链路更偏实用，不是强安全设计
- 简答题等更多题型支持还不完整
- 规则导入虽然已经增强，但仍然不可能覆盖所有来源格式
- 测试与工程化程度有限
