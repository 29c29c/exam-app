# 阶段 5：AI Markdown 题库导入增强

## 1. 阶段目标

本阶段聚焦“AI 生成题库内容”的导入体验，目标不是解析任意 Markdown 语法，而是稳定支持常见 AI 输出的 Markdown 题库文本。

本阶段完成后，应达到以下结果：

- 支持粘贴常见 AI 输出的 Markdown / 文本题库
- 在不破坏现有纯文本导入的前提下，提升 AI 题库识别率
- 通过“预清洗 + 结构化解析”两阶段策略控制误判
- 保持现有导入接口和结果结构基本稳定

## 2. 本阶段边界

本阶段要做：

- 新增 Markdown 预清洗能力
- 扩展题目、选项、答案、解析的识别规则
- 为导入接口增加 `sourceType`
- 增加 Markdown 预处理失败后的降级回退
- 补齐相关单元测试
- 更新导入提示文案和 README 说明

本阶段不做：

- 任意 Markdown 全兼容
- 表格型题库作为主输入格式
- AI 二次纠错或大模型兜底解析
- 为返回结构增加新的公开字段

## 3. 设计原则

### 3.1 支持目标

默认定位为“覆盖常见 AI 输出”，优先兼容以下常见风格：

- ChatGPT
- Claude
- Gemini
- DeepSeek

### 3.2 解析原则

采用“两阶段解析”：

1. 先做 Markdown 包装层剥离
2. 再复用现有题目识别规则，并补充少量面向 AI 输出的兼容规则

核心原则是“去包装，不改语义”，优先控制误判率，而不是无限放宽格式兼容。

## 4. 导入链路调整

### 4.1 新增 Markdown 预处理步骤

建议在导入解析链路中新增 `markdown preprocessor`，位置放在 `normalizeImportText` 之前，或作为其上游独立步骤。

推荐链路：

```text
raw input
  -> sourceType 判断
  -> Markdown 预处理（按需）
  -> normalizeImportText
  -> 题块切分
  -> 题型 / 选项 / 答案 / 解析提取
  -> 去重与预览结果
```

### 4.2 对外接口

保留现有接口：

- `POST /api/import/parse`

新增输入字段：

- `sourceType: 'markdown' | 'text' | 'auto'`

默认推荐：

- 前端传 `auto`

### 4.3 `auto` 模式识别特征

当输入文本具备以下 Markdown 特征时，优先进入 Markdown 预处理：

- 标题行 `#`
- 分隔线 `---` / `***`
- 列表项 `-` / `*` / `1.`
- 粗体包裹 `**答案**`
- 代码块围栏 ```` ``` ````
- 引用块 `>`
- 表格分隔符 `|---|`

## 5. Markdown 预处理规则

Markdown 预处理仅用于“去包装，不改语义”。

### 5.1 基础去包装

建议处理：

- 去掉标题层级标记 `#`，保留标题文本
- 去掉分隔线
- 去掉无意义列表符号
- 去掉引用前缀 `>`
- 去掉代码块围栏本身，保留代码块内部文本
- 去掉粗体 / 斜体包裹符号 `**`、`__`、`*`、`_`，保留正文

### 5.2 题号规范化

将常见列表题号统一成现有解析器更容易识别的格式，如 `1.`。

建议兼容：

- `- 1.`
- `1)`
- `## 第1题`
- `### 1.`
- `**1.**`
- `题目 1`
- `第 1 题`

### 5.3 选项规范化

将列表式选项规范成 `A. xxx` 形式。

建议兼容：

- `A. xxx`
- `A) xxx`
- `A：xxx`
- `- A. xxx`
- `* B) xxx`
- `**C.** xxx`

### 5.4 标签规范化

答案标签统一为标准答案标签，建议兼容：

- `答案`
- `参考答案`
- `正确答案`
- `Answer`
- `ANSWER`

解析标签统一为标准解析标签，建议兼容：

- `解析`
- `答案解析`
- `Explanation`
- `Analysis`

## 6. 结构化解析增强

### 6.1 题块起始识别

扩展题块起始识别规则，支持常见 AI Markdown 题头：

- `## 第1题`
- `### 1.`
- `**1.**`
- `题目 1`
- `第 1 题`

### 6.2 选项识别

扩展选项识别规则，支持：

- `A. xxx`
- `A) xxx`
- `A：xxx`
- `- A. xxx`
- `* B) xxx`
- `**C.** xxx`

### 6.3 答案标准化

多选答案建议统一为 `A,C` 格式，兼容：

- `A / C`
- `A、C`
- `A，C`
- `AC`

判断题答案建议统一为 `对/错`，兼容：

- `正确 / 错误`
- `是 / 否`
- `True / False`

## 7. 降级策略

为了避免 Markdown 规则过严导致“明明可导入却解析失败”，建议增加降级回退：

1. 先对预处理后的文本执行解析
2. 若可识别题数为 0，则自动回退到原始纯文本解析再试一次
3. 两次都失败时，继续返回现有错误提示“未能识别题目，请检查格式”

该策略的目标是：

- Markdown 输入尽量吃住
- 旧文本格式不被新规则误伤
- 失败时维持现有用户认知

## 8. 公共接口与数据行为

### 8.1 返回结构

解析结果结构保持不变：

- `previewId`
- `summary`
- `items`

### 8.2 原始文本保留策略

`items[*].rawText` 继续保留原始题块文本。

如果经过 Markdown 预处理，服务内部可额外保留 `normalizedRawText` 仅用于解析，但本期不修改对外返回结构。

## 9. 前端文案调整

前端导入弹窗文案建议更新为：

- “支持粘贴 AI 生成的 Markdown / 文本题库”

占位提示中建议加入推荐 Markdown 示例，但不改当前交互流程，不额外增加用户理解成本。

## 10. 测试计划

需要为 `services/import-parser.js` 和新增 Markdown 预处理逻辑补充单元测试，至少覆盖：

- 纯文本旧格式继续正常导入
- ChatGPT 风格：`## 第1题` + 粗体答案标签
- Claude 风格：标题 + 分隔线 + 列表选项
- Gemini / DeepSeek 常见风格：代码块包裹整套题目
- 多选答案写成 `A、C`、`A / C`、`AC`
- 判断题答案写成 `正确/错误`、`True/False`
- Markdown 噪音很多但题目正文完整时仍能识别
- 误判保护：普通说明文字、目录、总结段落不应被大量识别成题目
- 回退策略：Markdown 预处理失败但原始文本可识别
- 去重逻辑在 Markdown 与纯文本输入下表现一致

## 11. 验收标准

本阶段完成后，应满足：

- 现有文本导入测试全部通过
- 常见 AI Markdown 样例可识别率明显高于当前版本
- 不因新增兼容规则显著增加“把说明文本误识别成题”的情况
- 前端默认使用 `auto` 模式，用户无需理解额外解析选项
- README 和导入说明同步更新

## 12. 建议拆分任务

1. 为导入接口增加 `sourceType`
2. 实现 Markdown 特征检测与 `auto` 分流
3. 增加 Markdown 预处理模块
4. 扩展题号、选项、答案、解析识别规则
5. 增加 Markdown 解析失败后的降级回退
6. 补充解析与误判保护测试
7. 更新导入弹窗文案和 README

## 13. 阶段完成标志

当下面条件满足时，可认为阶段 5 完成：

- 用户可以直接粘贴大多数常见 AI 生成 Markdown 题库完成预解析
- 纯文本导入链路保持兼容
- Markdown 兼容增强主要解决“AI 输出包装层”问题，而未显著放大误判风险
